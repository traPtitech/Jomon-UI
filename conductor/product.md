# Initial Concept

Jomon UI is the frontend application for the accounting support service "Jomon", designed to streamline financial processes for the Digital Creation Club traP. It serves as a centralized platform for managing budget requests and reimbursements.

## Target Users

- **Club Members:** Users who need to submit budget requests and apply for reimbursements for their activities.
- **Accountants/Treasurers:** Users responsible for reviewing, approving, and managing financial transactions and records.

## Core Goals

- **User-Friendly Interface:** To provide an intuitive and efficient experience for submitting and tracking expense applications.
- **Streamlined Workflow:** To optimize the approval and reimbursement process, reducing administrative burden for treasurers.
- **Financial Integrity:** (Implicit) To ensure accurate record-keeping and budget management through a structured system.

## Key Features

- **Application Submission:** Comprehensive forms for submitting budget requests and reimbursement applications.
- **Approval Workflow:** A robust system for tracking application status, enabling communication via comments, and managing file attachments (receipts, etc.).

## Current Development Scope

- **Refactoring:** Enhancing type safety and code maintainability (specifically targeting `any` types and validation logic).
- **Bug Fixes:** addressing issues in the current application workflow to improve stability.
